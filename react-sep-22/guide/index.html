Making Components Reusable with Props

In our last lesson, we created a ProfileCard component. But it was hardcoded‚Äîit could only ever display Ada Lovelace's information. That's not very reusable! We need a way to pass data *into* our components. This is where **props** come in.

üí° The Custom Order Analogy

Think of a component like a coffee shop barista. The basic function is "make coffee." But you, the customer (the parent component), provide specific instructions: "I want a large latte with oat milk." Those instructions‚Äî"large," "latte," "oat milk"‚Äîare the **props**. They customize the final product without changing the fundamental process of making coffee.

Step 1: Passing Props from the Parent

In the parent component (App.jsx), we can pass props to our ProfileCard just like we would add attributes to an HTML tag.


// src/App.jsx
import ProfileCard from './ProfileCard';
import adaImage from './assets/ada.png';
import graceImage from './assets/grace.png';

function App() {
  return (
    <div>
      {/* We are passing 'name', 'title', and 'image' as props */}
      <ProfileCard 
        name="Ada Lovelace" 
        title="Mathematician & Writer" 
        image={adaImage} 
      />
      <ProfileCard 
        name="Grace Hopper" 
        title="Computer Scientist" 
        image={graceImage} 
      />
    </div>
  );
}

export default App;

Step 2: Receiving and Using Props in the Child

The child component receives all these props as a single object. By convention, we just call this object props. We can then use the data from this object in our JSX.


// src/ProfileCard.jsx
import React from 'react';
import './ProfileCard.css';

// The component receives a 'props' object as its first argument
function ProfileCard(props) {
  return (
    // We use curly braces {} to access the JavaScript values from the props object
    <div className="card">
        <img className="card-image" src={props.image} alt={props.name} />
        <h2 className="card-name">{props.name}</h2>
        <p className="card-title">{props.title}</p>
    </div>
  );
}

export default ProfileCard;

Just like that, our ProfileCard is now a truly reusable component that can display information for anyone!

Making Components Intelligent: Conditional Rendering üß†

What if you want to show something on the screen only if a certain condition is met? For example, show a "Login" button for guests but a "Logout" button for users. This is **conditional rendering**.

üí° The Light Switch Analogy

Conditional rendering is like a light switch. If the switch is **on** (the condition is true), you see the light (the UI). If the switch is **off** (the condition is false), the light is gone (the UI is not rendered).

Common Patterns for Conditional Rendering

1. Ternary Operator

Perfect for simple if/else logic. condition ? <ShowThis /> : <ShowThat />


function Greeting({ isLoggedIn }) {
  return (
    <h1>
      {isLoggedIn ? 'Welcome back!' : 'Please sign up.'}
    </h1>
  );
}

2. Logical && Operator

Perfect for rendering something *only* if a condition is true, otherwise rendering nothing. condition && <ShowThis />


function Mailbox({ unreadMessages }) {
  return (
    <div>
      
Your Mail

      {unreadMessages.length > 0 &&
        <p>
          You have {unreadMessages.length} unread messages.
        </p>
      }
    </div>
  );
}

Example: Adding a "Featured" Badge to our Profile Card

Let's add a boolean prop isFeatured. If it's true, we'll display a special badge on the card.


// In App.jsx, we pass the new prop to one of the cards
<ProfileCard name="Ada Lovelace" ... isFeatured={true} />
<ProfileCard name="Grace Hopper" ... isFeatured={false} />

// In ProfileCard.jsx, we use the && operator to conditionally render the badge
function ProfileCard(props) {
  return (
    <div className="card">
      {/* This div will ONLY render if props.isFeatured is true */}
      {props.isFeatured && <div className="featured-badge">‚≠êÔ∏è Featured</div>}
      
      <img className="card-image" src={props.image} alt={props.name} />
      <h2 className="card-name">{props.name}</h2>
      <p className="card-title">{props.title}</p>
    </div>
  );
}

üß† The "Aha!" Moment: The Heart of React üß†

You've just learned the two concepts that form the core of building applications in React.

Props make your components reusable and configurable, allowing you to build a design system of LEGO bricks.

Conditional Rendering makes your components dynamic and intelligent, allowing them to react to data and user interactions.

By combining these two superpowers, you can build anything from a simple button to a complex, data-driven dashboard. You are no longer just building static pages; you are building real, interactive applications!
