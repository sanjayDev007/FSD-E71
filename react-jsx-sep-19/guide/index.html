Why Convert HTML to React?

You've built a beautiful static webpage with HTML and CSS. So, why rebuild it in React? React transforms your static page into a dynamic, interactive application that is easier to manage, scale, and update.

üí° The LEGO Analogy

A static HTML page is like a single, solid LEGO sculpture. It looks great, but you can't easily change one part without affecting the whole thing.

A React app is like building that same sculpture out of individual, reusable LEGO bricks (Components). You can easily swap a red brick for a blue one, use the same "window" brick in multiple places, and manage the entire structure more effectively.

Step 1: The "Before" - Our Static HTML & CSS

Let's start with a simple profile card. This is our goal, the static version we want to convert.

index.html


<!DOCTYPE html>
<html lang="en">
<head>
    <link rel="stylesheet" href="style.css">
    <title>Profile Card</title>
</head>
<body>
    <div class="card">
        <img class="card-image" src="./profile.png" alt="User profile">
        <h2 class="card-name">Ada Lovelace</h2>
        <p class="card-title">Mathematician & Writer</p>
    </div>
</body>
</html>

style.css


.card {
  width: 300px;
  border: 1px solid #ccc;
  border-radius: 10px;
  box-shadow: 2px 2px 8px rgba(0,0,0,0.1);

Step 2: Thinking in React - Componentization üß†

First, let's set up a new React project using Vite (our recommended tool!). Open your terminal and run npm create vite@latest.

Now, look at our HTML and break it down into logical, reusable pieces. Our entire profile card can be a single component.

Inside our new Vite project, let's create a file: src/ProfileCard.jsx.

Step 3: Creating and Exporting the Component

We will now translate our HTML into JSX inside our new component file. There are a few key differences:

  • The class attribute in HTML becomes className in JSX.
  • We need to import images.
  • We export default the component so other files can use it.

src/ProfileCard.jsx


// 1. Import React and any assets like images and CSS
import React from 'react';
import profilePic from './assets/profile.png'; // Assuming you placed the image in src/assets
import './ProfileCard.css'; // We will create this CSS file next

// 2. Create a function component that returns JSX
function ProfileCard() {
  return (
    <div className="card">
        <img className="card-image" src={profilePic} alt="User profile" />
        <h2 className="card-name">Ada Lovelace</h2>
        <p className="card-title">Mathematician & Writer</p>
    </div>
  );
}

// 3. Export the component to make it available for import elsewhere
export default ProfileCard;

Step 4: Handling CSS in React

The simplest way to add CSS is to create a corresponding CSS file and import it directly into your component file, as shown above. Create src/ProfileCard.css and copy the CSS from our static example into it.

üí° Scoped Styles with CSS Modules

Importing CSS like this makes it global. For larger apps, class names can clash. A better way is using **CSS Modules**. If you rename your file to ProfileCard.module.css, React will automatically make your class names unique. You would then use them like this:


import styles from './ProfileCard.module.css';

// ...
<div className={styles.card}> 
  {/* The 'card' class is now unique to this component! */}
</div>

Step 5: Importing and Using Your Component

Now we just need to use our shiny new component! We do this by importing it into our main application file, src/App.jsx, and rendering it like an HTML tag.

src/App.jsx


import React from 'react';
import ProfileCard from './ProfileCard'; // Import our component
import './App.css'; // Main app styles

function App() {
  // Now we can use our component as if it were a standard HTML tag!
  return (
    <div className="App">
      <h1>My Awesome React App</h1>
      <ProfileCard />
    </div>
  );
}

export default App;

üß† The "Aha!" Moment: Reusability is Power üß†

You've successfully converted a static page into a React component! But what's the real magic? **Reusability.**

What if you wanted to display three profile cards? In HTML, you'd copy and paste the entire block of code three times. In React, you just reuse the component. This is the core philosophy that makes React so powerful for building complex applications. 


// Imagine doing this in App.jsx...
<div className="App">
  <ProfileCard />
  <ProfileCard />
  <ProfileCard />
</div>
// ...and you instantly get three profile cards!
