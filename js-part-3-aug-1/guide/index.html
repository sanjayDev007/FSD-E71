The Power of Expressions

In JavaScript, an operator is a symbol that performs an operation on one or more values (operands). An expression is any valid unit of code that resolves to a value. Combining variables with operators creates expressions!

üç≥ Analogy: The Recipe

Think of variables as your ingredients (eggs, flour) and operators as your actions (+ for mixing, > for comparing size). The entire instruction, like "mix 2 eggs + 1 cup of flour," is an expression. The final result is a delicious pancake batter (the resolved value!).

Meet the Operators

Let's look at the most common operators you'll use to make decisions in your code.

Arithmetic Operators

For doing math!

let sum = 10 + 5; // 15
let product = 10 * 5; // 50
let remainder = 10 % 3; // 1

Comparison Operators

For comparing values. They always result in a boolean (true or false).

let isGreater = 10 > 5; // true
// Strict equality (checks value AND type)
let isSame = 10 === "10"; // false

Logical Operators

For combining boolean expressions.

// AND (both must be true)
let result1 = (5 > 3) && (10 > 5); // true

// OR (at least one must be true)
let result2 = (5 < 3) || (10 > 5); // true

Making Decisions: if...else

The most common way to control the flow of your program is with an if...else if...else statement. It checks a condition and runs a specific block of code if that condition is true.

üõ£Ô∏è The Fork in the Road

This structure is like a series of forks in the road. The program checks the first path (if). If it's the right way, it takes it. If not, it checks the next path (else if), and so on, until it finds a match or takes the default path (else). 

function getTicketPrice(day) {
    if (day === "Saturday" || day === "Sunday") {
        return "Weekend price: $15";
    } else if (day === "Wednesday") {
        return "Discount Wednesday: $10";
    } else {
        return "Weekday price: $12";
    }
}

console.log(getTicketPrice("Saturday")); // "Weekend price: $15"
console.log(getTicketPrice("Monday"));   // "Weekday price: $12"

This works perfectly fine, but when you have many else if conditions checking the same variable, it can get a bit messy.

A Cleaner Path: The switch Statement

A switch statement is a cleaner alternative to a long if...else if chain when you are comparing a single value against multiple possible matches.

‚ùå Messy: The if...else if Chain

This code is repetitive. We keep writing day === ... over and over. It's readable, but we can do better!

function getFruitColor(fruit) {
  if (fruit === "apple") {
    return "Red or Green";
  } else if (fruit === "banana") {
    return "Yellow";
  } else if (fruit === "grape") {
    return "Purple";
  } else if (fruit === "orange") {
    return "Orange";
  } else {
    return "Unknown color";
  }
}

‚úÖ Clean: The switch Statement

This is much neater! We state the variable to check (fruit) once. Then, we list each case. The break keyword is crucial‚Äîit stops the code from "falling through" to the next case. The default case acts like the final else.

function getFruitColor(fruit) {
  switch (fruit) {
    case "apple":
      return "Red or Green";
    case "banana":
      return "Yellow";
    case "grape":
      return "Purple";
    case "orange":
      return "Orange";
    default:
      return "Unknown color";
  }
}

ü§Ø The Ultimate "Aha!" Moment

Use if...else for complex, varied conditions. Use switch for checking a single variable against a list of simple, direct matches.
Choosing the right tool makes your code easier to read, understand, and maintain. A switch statement clearly signals your intent: "Based on the value of this one thing, do one of these actions."
