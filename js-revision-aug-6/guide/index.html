Let's Build a Classic Game!

Today, we're going to build the timeless game of Rock, Paper, Scissors using JavaScript. This project is a fantastic way to practice fundamental concepts like getting user input, generating random numbers, and using conditional logic to determine a winner. We'll also learn a clever way to manage the game's rules using a JavaScript object as a hash map.

The Game Plan

Our game will follow these simple steps:

  1. Ask the player for their choice (rock, paper, or scissors).
  2. Make the computer randomly select a choice.
  3. Compare the two choices to see who wins.
  4. Display the result to the player.
üìñ Analogy: The Game's Rulebook (A Hash Map)

How do we know that paper beats rock? We could write a long, complicated if statement: if (player === 'paper' && computer === 'rock') .... But what if we add more options like "Lizard" and "Spock"? The `if` statements would become a nightmare!

Instead, we'll use a JavaScript object as a "rulebook" or hash map. It will instantly tell us what any given choice beats. For example, our rulebook will say: the key `"rock"` has a value of `"scissors"`. This means "rock beats scissors." It's a clean, fast, and scalable way to store our game's logic.

Let's Code! The Complete Game

For this simple game, we can put everything into a single HTML file. The magic happens inside the <script> tag. You can copy this code, save it as game.html, and open it in your browser to play!

The Full Code

// This function contains the entire game logic
function playGame() {
    // === STEP 1: DEFINE OUR GAME DATA ===

    // An array of possible choices for the computer
    const choices = ['stone', 'paper', 'scissors'];

    // The "Rulebook" Hash Map. The key beats its value.
    const rules = {
        stone: 'scissors',
        paper: 'stone',
        scissors: 'paper'
    };

    // === STEP 2: GET PLAYER AND COMPUTER CHOICES ===

    // Get player's choice using a prompt, and make it lowercase
    let playerChoice = prompt("Enter your choice: stone, paper, or scissors");

    // Basic validation: if the user cancels or enters nothing, stop the game.
    if (!playerChoice) {
        alert("Looks like you changed your mind. See you next time!");
        return; // Exit the function
    }
    
    playerChoice = playerChoice.toLowerCase();

    // Check if the player's choice is valid
    if (!choices.includes(playerChoice)) {
        alert("Oops! That's not a valid choice. Please choose stone, paper, or scissors.");
        return; // Exit the function
    }

    // Generate a random index to pick the computer's choice
    const randomIndex = Math.floor(Math.random() * choices.length);
    const computerChoice = choices[randomIndex];

    // === STEP 3: DETERMINE THE WINNER ===

    let resultMessage = `You chose ${playerChoice}. The computer chose ${computerChoice}.\n`;

    // Use an if/else if/else chain to check the three possible outcomes
    if (playerChoice === computerChoice) {
        // 1. Check for a tie
        resultMessage += "It's a tie!";
    } else if (rules[playerChoice] === computerChoice) {
        // 2. Check if the player wins using our rulebook
        // e.g., if player chose 'paper', rules['paper'] is 'stone'. If computer chose 'stone', player wins.
        resultMessage += "You win! üéâ";
    } else {
        // 3. If it's not a tie and the player didn't win, the computer must have won.
        resultMessage += "The computer wins! ü§ñ";
    }

    // === STEP 4: DISPLAY THE RESULT ===
    alert(resultMessage);
}

// Start the game when the script loads!
playGame();

ü§Ø The Power of the Hash Map

Look at the winning condition: rules[playerChoice] === computerChoice.

This one line replaces a complex set of `if` statements. It's readable, efficient, and if you ever wanted to add more rules, you would just add a new entry to the rules object instead of writing more `else if` blocks. This is a perfect example of how choosing the right data structure can make your code dramatically simpler and better.
