1. THE <html> ELEMENT - THE FOUNDATION
================================================================================

Purpose: Root container for the entire webpage
Analogy: The big circus tent that contains everything

Essential Attributes:
- lang="en"          Language declaration (helps SEO & accessibility)
- dir="ltr"          Text direction (ltr = left-to-right, rtl = right-to-left)

Common Language Codes:
- lang="en"          English
- lang="es"          Spanish  
- lang="fr"          French
- lang="de"          German
- lang="ar"          Arabic (use with dir="rtl")

Why It Matters:
‚úì Search engines use language info for ranking
‚úì Screen readers understand content better
‚úì Browsers can offer translation services
‚úì Required for valid HTML5

Example:
<html lang="en" dir="ltr">
    <!-- Everything else goes inside -->
</html>

================================================================================
2. THE <head> ELEMENT - THE BRAIN
================================================================================

Purpose: Contains metadata (info FOR browsers, not visible to users)
Analogy: Backstage area of a theater - crucial but hidden from audience

Essential Elements in <head>:

1. CHARACTER ENCODING
   <meta charset="UTF-8">
   - Supports all languages and emojis üåç
   - Prevents weird character display
   - MUST be first meta tag

2. VIEWPORT (Mobile Responsiveness)
   <meta name="viewport" content="width=device-width, initial-scale=1.0">
   - Makes website work properly on mobile devices
   - Controls zoom and scaling behavior

3. PAGE TITLE
   <title>Your Page Title</title>
   - Shows in browser tabs
   - Used by search engines for results
   - Used when bookmarking pages
   - Keep under 60 characters for SEO

4. SEO META TAGS
   <meta name="description" content="Brief page description">
   <meta name="keywords" content="keyword1, keyword2, keyword3">
   <meta name="author" content="Your Name">

5. CSS STYLESHEETS
   <link rel="stylesheet" href="styles.css">
   - Links external CSS files
   - Better performance than inline styles

6. FAVICON (Website Icon)
   <link rel="icon" type="image/x-icon" href="/favicon.ico">
   - Small icon in browser tab
   - Professional touch

7. FONTS
   <link rel="preconnect" href="https://fonts.googleapis.com">
   <link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;600&display=swap" rel="stylesheet">

Complete <head> Example:
<head>
    <!-- Essential meta tags -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    
    <!-- SEO -->
    <title>HTML Basics - Learn Web Development</title>
    <meta name="description" content="Master HTML fundamentals with clear examples">
    <meta name="keywords" content="HTML, web development, tutorial">
    <meta name="author" content="Your Name">
    
    <!-- Favicon -->
    <link rel="icon" type="image/x-icon" href="/favicon.ico">
    
    <!-- CSS -->
    <link rel="stylesheet" href="css/styles.css">
    <link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;600&display=swap" rel="stylesheet">
</head>

================================================================================
3. THE <body> ELEMENT - THE SHOW
================================================================================

Purpose: Contains all visible content that users see and interact with
Analogy: Main stage where the performance happens

Common Content Types:

1. HEADINGS (h1-h6)
   <h1>Main Title</h1>         (Most important - only one per page)
   <h2>Section Title</h2>       
   <h3>Subsection</h3>         
   <!-- h4, h5, h6 for smaller headings -->

2. PARAGRAPHS
   <p>This is a paragraph of text content.</p>

3. LINKS
   <a href="https://example.com">Visit Example</a>        (External link)
   <a href="#section1">Go to Section 1</a>               (Internal link)
   <a href="mailto:email@example.com">Send Email</a>      (Email link)

4. IMAGES
   <img src="photo.jpg" alt="Description of image">
   - alt attribute is required for accessibility
   - Describes image for screen readers

5. LISTS
   Unordered (bullet points):
   <ul>
       <li>Item 1</li>
       <li>Item 2</li>
   </ul>
   
   Ordered (numbered):
   <ol>
       <li>First item</li>
       <li>Second item</li>
   </ol>

6. LAYOUT CONTAINERS
   <div class="container">Content here</div>      (Generic container)
   <section>Distinct content section</section>    (Semantic section)
   <article>Independent content</article>         (Standalone content)

7. SEMANTIC STRUCTURE
   <header>Site header, logo, navigation</header>
   <nav>Navigation menu links</nav>
   <main>Primary page content</main>
   <aside>Sidebar content</aside>
   <footer>Contact info, copyright</footer>

Typical <body> Structure:
<body>
    <header>
        <h1>Site Title</h1>
        <nav>
            <a href="#about">About</a>
            <a href="#contact">Contact</a>
        </nav>
    </header>
    
    <main>
        <section id="about">
            <h2>About Section</h2>
            <p>Content here...</p>
        </section>
        
        <section id="contact">
            <h2>Contact Section</h2>
            <p>More content...</p>
        </section>
    </main>
    
    <footer>
        <p>&copy; 2025 Your Website</p>
    </footer>
</body>

COMMON MISTAKES TO AVOID
================================================================================

‚ùå WRONG:
- Forgetting <!DOCTYPE html>
- Missing lang attribute: <html>
- Missing charset: <head><title>Page</title></head>
- Missing viewport meta tag
- Putting visible content in <head>: <head><h1>Title</h1></head>
- Multiple <h1> tags on same page

‚úÖ CORRECT:
- Always start with <!DOCTYPE html>
- Always include lang: <html lang="en">
- Always include charset: <meta charset="UTF-8">
- Always include viewport: <meta name="viewport" content="width=device-width, initial-scale=1.0">
- All visible content goes in <body>
- Only one <h1> per page (for SEO)
