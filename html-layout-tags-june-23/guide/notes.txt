üèóÔ∏è OVERVIEW - THE HTML5 SEMANTIC LAYOUT FAMILY
================================================================================

The HTML5 semantic layout tags provide meaningful structure to web pages:
- They define the PURPOSE and RELATIONSHIP of different content areas
- They improve accessibility for screen readers and assistive technologies
- They enhance SEO by helping search engines understand content structure
- They create self-documenting code that's easier to maintain

ANALOGY: Think of these tags as rooms in a smart house:
- header = entrance with welcome sign and house number
- nav = hallway that helps you navigate between rooms
- main = living area where primary activities happen
- sections = different themed rooms (kitchen, bedroom, etc.)
- articles = individual furniture pieces with complete functionality
- footer = basement with utilities and foundation information

üéØ HEADER TAG - THE DIGITAL FRONT DOOR
================================================================================

PURPOSE: Contains introductory content, site branding, and main navigation

WHAT IT SHOULD CONTAIN:
‚úÖ Site logo/brand name
‚úÖ Main site navigation
‚úÖ Site tagline/description
‚úÖ Contact information (optional)
‚úÖ Search functionality (optional)

BEST PRACTICES:
- Only use ONE header per page (for main site header)
- Can also be used within articles/sections for their headers
- Should be the first thing users see
- Must work on all device sizes (responsive)
- Include proper heading hierarchy (h1 for main title)

EXAMPLE - GOOD:
<header>
    <h1>Amazing Company</h1>
    <p>Your trusted web partner</p>
    <nav>
        <ul>
            <li><a href="/">Home</a></li>
            <li><a href="/about">About</a></li>
            <li><a href="/contact">Contact</a></li>
        </ul>
    </nav>
</header>

EXAMPLE - BAD:
<div class="top-area">
    <div class="logo">Site Name</div>
    <div class="menu">
        <a href="#">Link 1</a>
        <a href="#">Link 2</a>
    </div>
</div>

COMMON MISTAKES:
‚ùå Using generic divs instead of semantic header
‚ùå Multiple main headers on one page
‚ùå Poor mobile responsiveness
‚ùå Missing or unclear navigation

üß≠ NAVIGATION TAG - THE SITE COMPASS
================================================================================

PURPOSE: Contains navigation links that help users move around the site

TYPES OF NAVIGATION:
1. Main navigation (primary site menu)
2. Breadcrumb navigation (shows current location)
3. Pagination (for multi-page content)
4. Footer navigation (secondary links)

BEST PRACTICES:
- Use aria-label to distinguish multiple nav elements
- Include current page indicators (aria-current="page")
- Organize links logically
- Make touch-friendly for mobile
- Use meaningful link text (not "click here")

EXAMPLE - EXCELLENT:
<nav aria-label="Main navigation">
    <ul>
        <li><a href="/" aria-current="page">Home</a></li>
        <li><a href="/about">About Us</a></li>
        <li><a href="/services">Services</a></li>
        <li><a href="/contact">Contact</a></li>
    </ul>
</nav>

<nav aria-label="Breadcrumb">
    <ol>
        <li><a href="/">Home</a></li>
        <li><a href="/products">Products</a></li>
        <li>Laptops</li>
    </ol>
</nav>

ACCESSIBILITY FEATURES:
‚úÖ aria-label for multiple nav elements
‚úÖ aria-current for current page
‚úÖ Keyboard navigation support
‚úÖ Screen reader friendly structure

COMMON MISTAKES:
‚ùå Using divs instead of nav
‚ùå Multiple nav elements without context
‚ùå Poor organization/labeling
‚ùå Unclear current state

üé™ MAIN TAG - THE STAR OF THE SHOW
================================================================================

PURPOSE: Contains the dominant content of the page

KEY RULES:
- Only ONE main tag per page
- Should contain the primary content
- Excludes headers, footers, and sidebars
- Perfect target for "skip to main content" links

WHAT IT SHOULD CONTAIN:
‚úÖ Page's primary content
‚úÖ Main heading (h1)
‚úÖ Sections and articles
‚úÖ Core functionality/information

EXAMPLE - CORRECT:
<body>
    <header>
        <h1>Site Name</h1>
        <nav>...</nav>
    </header>
    
    <main>
        <h1>Page Title</h1>
        <section>...</section>
        <section>...</section>
    </main>
    
    <footer>...</footer>
</body>

EXAMPLE - WRONG:
<body>
    <header>...</header>
    
    <main>
        <h1>Welcome</h1>
        <p>Some content...</p>
    </main>
    
    <main> <!-- ‚ùå Second main tag! -->
        <h2>More content</h2>
    </main>
    
    <footer>...</footer>
</body>

BENEFITS:
‚úÖ Clear content hierarchy
‚úÖ Perfect accessibility
‚úÖ SEO optimization
‚úÖ Skip navigation target

üìÇ SECTION TAG - THE ORGANIZER
================================================================================

PURPOSE: Groups related content under a common theme

WHEN TO USE:
‚úÖ Content has a clear theme/topic
‚úÖ You would add it to a table of contents
‚úÖ It typically has a heading
‚úÖ Grouping related articles or content

CHARACTERISTICS:
- Usually starts with a heading (h2, h3, etc.)
- Contains thematically related content
- Can contain articles, paragraphs, images, etc.
- Creates logical document outline

EXAMPLE - GOOD:
<section>
    <h2>About Us</h2>
    <p>Company history and mission...</p>
    <p>Our values and goals...</p>
</section>

<section>
    <h2>Our Team</h2>
    <article>Team member 1...</article>
    <article>Team member 2...</article>
</section>

EXAMPLE - BAD:
<div class="content-area">
    <h2>About Us</h2>
    <p>Content...</p>
    <h2>Our Team</h2>
    <p>Different content...</p>
</div>

SECTION VS DIV:
- Use SECTION when content is thematically related
- Use DIV for purely styling/layout purposes
- SECTION has semantic meaning, DIV does not

üì∞ ARTICLE TAG - THE SELF-CONTAINED STAR
================================================================================

PURPOSE: Contains standalone, self-contained content

THE "SYNDICATION TEST":
Could this content be:
- Removed from the page and still make sense?
- Published on another website?
- Distributed in an RSS feed?
- Read independently?

If YES ‚Üí use <article>
If NO ‚Üí use <section> or <div>

PERFECT FOR:
‚úÖ Blog posts
‚úÖ News articles
‚úÖ Product descriptions
‚úÖ User comments
‚úÖ Forum posts
‚úÖ Social media posts

STRUCTURE:
<article>
    <header>
        <h2>Article Title</h2>
        <p>By Author, Date</p>
    </header>
    
    <p>Article content...</p>
    
    <section>
        <h3>Subsection</h3>
        <p>More content...</p>
    </section>
    
    <footer>
        <p>Tags, categories, etc.</p>
    </footer>
</article>

COMMON MISTAKES:
‚ùå Using article for navigation menus
‚ùå Using article for copyright notices
‚ùå Using article for content that's not standalone
‚ùå Confusing article with section

üè† FOOTER TAG - THE FOUNDATION
================================================================================

PURPOSE: Contains footer information for a page or section

WHAT IT SHOULD CONTAIN:
‚úÖ Copyright information
‚úÖ Contact details
‚úÖ Links to privacy policy, terms
‚úÖ Social media links
‚úÖ Secondary navigation
‚úÖ Company information
‚úÖ Site map links

BEST PRACTICES:
- Include comprehensive contact information
- Organize content in logical sections
- Use proper navigation structure for footer links
- Make it visually distinct but accessible
- Include important legal information

EXAMPLE - COMPREHENSIVE:
<footer>
    <section>
        <h3>Contact Us</h3>
        <p>Email: hello@company.com</p>
        <p>Phone: (555) 123-4567</p>
        <p>Address: 123 Tech Street</p>
    </section>
    
    <section>
        <h3>Quick Links</h3>
        <nav>
            <ul>
                <li><a href="/privacy">Privacy Policy</a></li>
                <li><a href="/terms">Terms of Service</a></li>
                <li><a href="/support">Support</a></li>
            </ul>
        </nav>
    </section>
    
    <section>
        <h3>Follow Us</h3>
        <p>Twitter | Facebook | Instagram</p>
    </section>
    
    <p>&copy; 2024 Company Name. All rights reserved.</p>
</footer>

FOOTER SCOPING:
- Page footer: Contains site-wide information
- Article footer: Contains article-specific information (tags, author, etc.)
- Section footer: Contains section-specific information

üö´ COMMON MISTAKES TO AVOID
================================================================================

1. MULTIPLE MAIN TAGS
   ‚ùå <main>...</main><main>...</main>
   ‚úÖ Only one <main> per page

2. WRONG TAG USAGE
   ‚ùå <article> for navigation or copyright
   ‚úÖ <article> only for standalone content

3. GENERIC CONTAINERS
   ‚ùå <div class="header">
   ‚úÖ <header>

4. POOR HEADING HIERARCHY
   ‚ùå <h1> ‚Üí <h4> (skipping h2, h3)
   ‚úÖ <h1> ‚Üí <h2> ‚Üí <h3> (logical order)

5. MISSING ACCESSIBILITY
   ‚ùå <nav><nav> without context
   ‚úÖ <nav aria-label="Main navigation">

6. OVERUSING SECTIONS
   ‚ùå <section> for every div
   ‚úÖ <section> only for thematic groups

7. MISSING NAVIGATION STRUCTURE
   ‚ùå <nav><a href="#"></nav>
   ‚úÖ <nav><ul><li><a href="#"></nav>

üéØ QUICK REFERENCE GUIDE
================================================================================

TAG         PURPOSE                 WHEN TO USE
----------- ----------------------- ------------------------------------------
<header>    Site/content header     Site branding, navigation, intro content
<nav>       Navigation links        Menus, breadcrumbs, pagination
<main>      Primary content         Main page content (ONE per page)
<section>   Thematic groups         Related content under common theme
<article>   Standalone content      Blog posts, news, comments, reviews
<footer>    Footer information      Contact, legal, secondary links

NESTING RELATIONSHIPS:
- <body> ‚Üí <header>, <main>, <footer>
- <main> ‚Üí <section>, <article>
- <section> ‚Üí <article>, <p>, <h2-h6>
- <article> ‚Üí <header>, <section>, <footer>, content
- <header>/<footer> ‚Üí can contain <nav>

ACCESSIBILITY ATTRIBUTES:
- aria-label: Distinguish multiple nav elements
- aria-current="page": Mark current page in navigation
- role="main": Additional ARIA role for main (optional)
